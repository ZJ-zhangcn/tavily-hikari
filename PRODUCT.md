# Product

## Register

product

## Users

Tavily Hikari is used by self-hosting operators, administrators, and trusted end users who need to route Tavily MCP and HTTP traffic through a managed key pool. They work in operational contexts: checking service health, managing upstream keys, reviewing request logs, issuing access tokens, and copying integration snippets into developer tools.

## Product Purpose

The product provides a privacy-aware proxy and web console for Tavily access. Success means operators can understand key health and traffic behavior quickly, while users can find their token, quota, and setup instructions without exposing real Tavily secrets.

## Brand Personality

Tactile, optimistic, and precise. The interface should feel friendly and memorable without reducing trust in the operational surfaces.

## Anti-references

Avoid generic dark observability dashboards, flat shadcn defaults, noisy toy-like admin tables, low-contrast pastel text, sharp enterprise chrome, and decorative motion that slows task completion.

## Design Principles

- Make operational state legible first, then make it pleasant.
- Use the clay aesthetic as a tactile product language, not as decoration over data.
- Keep high-risk admin actions visually calm and structurally predictable.
- Make token and setup workflows feel guided, copyable, and safe.
- Preserve responsive usefulness across mobile console use and desktop admin review.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Respect reduced motion. Keep touch targets at least 44px where practical. Do not communicate status with color alone.
